# IsaacSimSharp — project summary

A reference you can carry into another project. Captures the architecture, the wire
protocol, the full API surface, the **verified** Isaac Sim 6.0 API facts, and the
non-obvious gotchas discovered by running against the real simulator.

Repo: https://github.com/CristianMori/IsaacSimSharp (Apache-2.0)
Target: NVIDIA Isaac Sim **6.0.0** (GA 2026), .NET **10** (`net10.0`).

---

## 1. What it is

A typed **C# SDK** that drives Isaac Sim 6.0 in realtime: configure a scene, control
robots, read sensors, manipulate any USD prim, and export USD. Isaac Sim's only supported
scripting surface is **Python in-process inside Omniverse Kit**, so the SDK talks to a small
**Python bridge** that runs inside Isaac Sim.

```
   C# app  ──►  IsaacSimSharp (SDK)
                     │  ZeroMQ + Protobuf  (localhost by default)
                     ▼
              isaacsim_bridge (Python, runs INSIDE Isaac Sim)
                     │  in-process calls
                     ▼
              Isaac Sim 6.0  (PhysX, RTX, USD, robots, sensors)
```

**Why Python in the bridge (not C++):** the bridge only dispatches commands and packs
buffers; the heavy compute (PhysX, RTX, CUDA) is compiled C++/GPU regardless of caller, so
Python adds negligible cost. The high-level robotics/sensor API (`isaacsim.core.experimental.*`,
RTX sensor wrappers, URDF importer) is documented/stable **only in Python**; a C++ Kit
extension is possible but must build against lower-level, less-documented PhysX/USD/Fabric
APIs. Python = fastest path to correct, riding NVIDIA's own supported surface and their
official `IsaacSimZMQ` ZeroMQ+Protobuf pattern. A future native runtime (ovphysx + ovrtx)
can consume the **exported USD** with no Python.

## 2. Transport & wire protocol

Two ZeroMQ sockets, one Protobuf schema (`proto/isaacsim.proto`) as the single source of truth.

- **Commands** — DEALER (C#) / ROUTER (bridge), request/reply correlated by `id`. A `Command`
  envelope holds a `oneof request`; the bridge returns `Reply { id, ok, error, oneof result }`.
- **Sensors** — PUB (bridge) / SUB (C#). The bridge publishes `SensorFrame` messages with the
  sensor handle as the topic; C# exposes them as `IAsyncEnumerable<SensorFrame>`.

Default endpoints: commands `tcp://127.0.0.1:5599`, sensors `tcp://127.0.0.1:5600`.
(Tests use a mock on `5611/5612`.) Localhost is the default but nothing hard-codes it —
it's a connection-string change for networked use.

**Bridge loop:** single-threaded, matching Isaac Sim's model. Each iteration: non-blocking
poll of the command socket → dispatch on the **sim thread** → `simulation_app.update()` →
publish frames for subscribed sensors. No cross-thread marshaling.

**C# transport:** a NetMQ background poller owns the sockets; outgoing requests map to
`TaskCompletionSource` by `id` (pipelined/async). SUB messages flow into a
`System.Threading.Channels` channel surfaced as `IAsyncEnumerable`.

**Codegen:** C# classes are generated at build by `Grpc.Tools` (messages only, no gRPC
services) **directly into the SDK project** (no separate Protocol assembly — that caused
phantom NuGet deps). Python classes are generated by `tools/gen_proto.ps1`, which reuses the
`protoc.exe` shipped inside the restored `Grpc.Tools` NuGet package (no separate protoc to
install). Generated `*_pb2.py` are git-ignored and regenerated. **protoc gotcha:** pass the
`.proto` file **relative to** `--proto_path` (bare filename, not an absolute path) or it errors
with "File does not reside within any path specified using --proto_path".

## 3. Repository layout

```
proto/isaacsim.proto              # wire protocol — single source of truth
src/IsaacSimSharp/                # the C# SDK (net10.0)
  IsaacSimClient.cs               # facade: lifecycle + Scene/Robots/Sensors/Usd/Physics + handle factories
  Transport/                      # NetMQ DEALER command channel + SUB sensor channel
  Scene/ Robots/ Sensors/ Usd/ Physics/   # typed wrappers
  Hosting/IsaacSimBridge.cs       # LaunchAsync — start/stop Isaac Sim from C#
  Imaging/Png.cs                  # dependency-free PNG encoder (no ImageSharp license issue)
bridge/isaacsim_bridge/           # Python bridge (runs inside Isaac Sim)
  __main__.py                     # SimulationApp(headless) → bind ZMQ → step loop
  handlers.py                     # ~60 handlers, one per oneof case; handle registry
  proto/                          # generated Python protobuf (git-ignored)
  run_bridge.bat                  # wraps C:\isaacsim\python.bat -m isaacsim_bridge
mock/mock_bridge.py               # pure-Python ZMQ server, NO Isaac Sim — fast C# dev/CI
tools/gen_proto.ps1               # regenerate Python protobuf from proto/
samples/                          # ~27 focused snippets + end-to-end samples + READMEs
tests/IsaacSimSharp.Tests/        # xUnit, runs against the mock (41 tests)
```

**Dispatch convention:** the bridge resolves a command via `getattr(self, f"_h_{which}")`
where `which = cmd.WhichOneof("request")`. To add an op: add it to the `oneof`, add a
`_h_<name>` handler, mirror it in the mock, add a C# wrapper + test.

## 4. C# API surface

```csharp
using var client = IsaacSimClient.Connect();          // or IsaacSimBridge.LaunchAsync(...) to self-host
```

- **Lifecycle** (`client`): `NewStageAsync`, `OpenStageAsync`, `PlayAsync`, `PauseAsync`,
  `StopAsync`, `ResetAsync`, `StepAsync`, `SetPhysicsDtAsync`, `ExportUsdAsync`, `ShutdownAsync`,
  `GetAssetsRootAsync`, `PingAsync`, `GetVersionAsync`.
- **Scene** (`client.Scene`): `AddGroundPlaneAsync`, `AddLightAsync`, `AddPrimitiveAsync`,
  `AddReferenceAsync`, `ImportUrdfAsync`, `SetPrimPoseAsync`, `RemovePrimAsync`.
- **Robots** (`client.Robots`): `RegisterAsync` → `RobotArticulation`:
  - `GetStateAsync` → `DofState(Positions, Velocities, Efforts)` — efforts are **commanded**.
  - `GetLinkForcesAsync` → `LinkForces(LinkNames, Forces[], Torques[])` — **sensed** per-link
    6D reaction (force feedback), `Vector3` each. Distinct from commanded efforts.
  - `SetPositionTargetsAsync`, `SetVelocityTargetsAsync`, `SetEffortsAsync` (optional `dofIndices`).
- **Sensors** (`client.Sensors`):
  - `CreateCameraAsync(path, w, h, position, orientation, depth=true, segmentation=false,
    instanceSegmentation=false, normals=false)` — annotators bundled into one `ImageFrame`.
  - `CreateImuAsync`, `CreateContactAsync`, `CreateLidarAsync(config="Example_Rotary")`,
    `CreateRadarAsync(config="IWRL6432AOP")` (needs `--motion-bvh`).
  - `GetFrameAsync(handle)` (pull, one-shot) and `StreamAsync(handle)` (push, `IAsyncEnumerable`,
    auto-subscribes/unsubscribes).
  - `DepthCloud.ToPoints(ImageFrame)` — deproject depth → camera-space `Vector3[]` using intrinsics.
- **USD** (`client.Usd`): reflective stage access — `ListPrimsAsync`/`FindPrimsAsync`,
  `GetPrimAsync`, `DefinePrimAsync` (any USD type), `GetAttributeAsync`/`SetAttributeAsync`,
  `GetTransformAsync`/`SetTransformAsync`/`GetBoundsAsync`, `SetVisibilityAsync`/`SetActiveAsync`,
  `ApplySchemaAsync`, `SetMassAsync`, `CreateMaterialAsync`/`BindMaterialAsync`,
  `MovePrimAsync` (reparent/rename), `DuplicatePrimAsync`.
- **Physics** (`client.Physics`): `SetRigidPoseAsync` (teleport), `SetVelocityAsync`/`GetVelocityAsync`,
  `RaycastAsync` (closest-hit scene query).
- **Handles / object paradigm**: `client.CreateCubeAsync` / `client.GetPrim` / `client.DefinePrimAsync`
  return `Prim`/`Cube` objects with async methods and a batched `EditAsync()` scope that flushes
  all local mutations in **one** round-trip:
  ```csharp
  var cube = await isaac.CreateCubeAsync("/World/Box");
  await cube.SetPositionAsync(1, 2, 3);
  await using (var e = await cube.EditAsync()) { e.Size.Width = 30; e.Position.Y += 4; } // one round-trip
  ```

## 5. Verified Isaac Sim 6.0 API facts (grounded on disk at C:\isaacsim)

- Entry: `from isaacsim import SimulationApp` → instantiate → **then** import everything else.
- Namespaces are `isaacsim.core.experimental.*`:
  - Stage: `...utils.stage` → `create_new_stage()`, `add_reference_to_stage(usd_path, path)`.
  - App/lifecycle: `...utils.app` → `play()`, `is_playing()`, `stop()`.
  - Objects: `...objects` → `Cube`, `Sphere`, `GroundPlane`, `DistantLight`, `Camera`.
  - Prims: `...prims` → `Articulation`, `RigidPrim`, `GeomPrim`.
  - Sim config: `isaacsim.core.simulation_manager.SimulationManager` → `set_physics_dt`,
    `set_physics_sim_device`.
- **No `ArticulationAction` in 6.0 experimental.** Drive/read directly on `Articulation`:
  `get_dof_positions()`, `get_dof_velocities()`, `get_dof_efforts()`,
  `set_dof_position_targets()`, `set_dof_velocity_targets()`, `set_dof_efforts()`.
  Values are `(N, D)` shaped (N=1 for a single articulation). `dof_names`/`num_links`/`link_names`
  may be unavailable before play.
- **Sensed joint forces:** `Articulation.get_link_incoming_joint_force()` → `(forces, torques)`,
  each shape `(N, L, 3)`. The root/base link reads **zero** (no incoming joint). Requires physics
  playing (tensor API).
- **Sensors:**
  - `isaacsim.sensors.experimental.rtx` → `RtxCamera`+`CameraSensor`, `Lidar`+`LidarSensor`,
    `Radar`+`RadarSensor`. `RtxCamera.camera` is the underlying `Camera` prim wrapper (intrinsics).
  - `isaacsim.sensors.experimental.physics` → `IMU`+`IMUSensor`, `Contact`+`ContactSensor`.
  - Lidar/radar use the `generic-model-output` annotator (GMO buffer); decode with
    `parse_generic_model_output_data` → azimuth/elevation/range → Cartesian.
- **Camera annotators** (`CameraSensor(cam, resolution=(h,w), annotators=[...])`), valid names:
  `rgb`, `rgba`, `distance_to_image_plane`, `distance_to_camera`, `instance_segmentation`,
  `instance_id_segmentation`, `semantic_segmentation`, `normals`, `motion_vectors`, `pointcloud`,
  `bounding_box_2d_loose/tight`, `bounding_box_3d`. Resolution is **(height, width)** (NumPy order).
- **USD export:** `omni.usd.get_context().get_stage().Export("path.usda")` (ASCII for diffable dev).
- **URDF import:** `isaacsim.asset.importer.urdf.impl`.
- **Bundled Python** has `numpy` + `warp` but **NOT** `pyzmq`/`protobuf` — install them:
  `C:\isaacsim\python.bat -m pip install -r bridge/requirements.txt`.
- Reference examples: `C:\isaacsim\standalone_examples\api\...`.

## 6. Sensor frame details (exact buffer layouts)

`SensorFrame` carries one of: `ImageFrame` (camera), `ImuFrame`, `ContactFrame`, `PointCloudFrame`
(lidar/radar). `ImageFrame` fields:

| field | layout | notes |
|---|---|---|
| `data` | `w*h*channels` bytes | rgba8 (4ch) or rgb8 (3ch) |
| `depth` | `w*h*4` bytes, float32 | `distance_to_image_plane` (meters, along optical axis) |
| `segmentation` | `w*h*4` bytes, uint32 | semantic label per pixel; `segmentation_labels` = id→label |
| `instance_segmentation` | `w*h*4` bytes, uint32 | `instance_labels` = id→prim path |
| `normals` | **`w*h*3*4`** bytes, float32 | **3 floats (x,y,z) per pixel**, NOT 4 — verified live |
| `intrinsics` | `CameraIntrinsics{fx,fy,cx,cy}` | pinhole, in pixels |

**Pinhole intrinsics** (bridge derives from focal length + aperture; units cancel):
```
fx = focal_length * width  / horizontal_aperture
fy = focal_length * height / vertical_aperture
cx = width  / 2
cy = height / 2
```
(A default 320×240 camera reports fx=fy≈763.54 from 50 mm focal / 20.955 mm aperture.)

**Depth → point cloud** (camera optical frame: +x right, +y down, +z forward):
```
x = (u - cx) * d / fx
y = (v - cy) * d / fy
z = d            # d = distance_to_image_plane
```
Skip pixels with non-finite or non-positive depth (sky/background).

## 7. Gotchas & lessons learned (the expensive-to-rediscover ones)

- **A freshly created camera produces no data until the timeline is playing AND several updates
  have elapsed.** `GetFrameAsync` right after `CreateCameraAsync` returns "no data available yet"
  — even for RGB. Call `PlayAsync()` then `StepAsync(~30-60)` to warm the RTX pipeline. The
  bridge's own startup does `WARMUP_FRAMES = 8` before printing "listening on".
- **`normals` returns 3 floats/pixel, not 4.** Assuming `w*h*4` over-allocates; it's `w*h*3*4`.
  Always confirm annotator buffer shapes against the live sim, not assumptions.
- **RTX radar crashes the sim natively unless Motion BVH is enabled.** Launch the bridge with
  `--motion-bvh` (`enable_motion_bvh` / `/renderer/raytracingMotion/enabled`). The bridge guards
  this and raises a clean error instead of crashing.
- **Constant depth over flat ground is correct, not a bug.** A downward-looking camera over a
  horizontal ground plane sees uniform `distance_to_image_plane` (the plane is perpendicular to
  the optical axis).
- **Long `StepAsync(N)` can exceed the request timeout.** The SDK chunks steps internally
  (`StepChunk = 60`) so a single large step doesn't time out.
- **First RTX update at cold start is slow** → warmup frames before announcing readiness.
- **NuGet packaging:** keep protobuf codegen inside the SDK project; a separate Protocol assembly
  produced a phantom dependency and shipped without the DLL.
- **PNG encoding:** ImageSharp v4 needs a commercial license → the repo ships a dependency-free
  PNG encoder (`IsaacSimSharp.Imaging.Png`).
- **Tests:** parallel xUnit classes each spinning a mock on the same port → "Address in use."
  Fix: one shared mock via `ICollectionFixture` (`MockBridgeCollection`) on dedicated ports;
  each test gets its own client via `_fixture.CreateClient()`.
- **Windows/bash:** launching `.bat` files through the bash tool mangled backslash-`r`
  (`bridge\run_bridge.bat` → `un_bridge.bat`). Launch via PowerShell `Start-Process cmd /c ...`
  or use forward slashes.

## 8. Running it

```powershell
# one-time: install bridge deps into Isaac Sim's bundled Python
C:\isaacsim\python.bat -m pip install -r bridge/requirements.txt   # pyzmq, protobuf

# launch the bridge inside Isaac Sim (headless by default)
bridge\run_bridge.bat --command-endpoint tcp://127.0.0.1:5599 --sensor-endpoint tcp://127.0.0.1:5600
#   --gui          show the Isaac Sim window
#   --motion-bvh   required for RTX radar (Doppler)
#   --livestream   WebRTC stream on localhost:49100 (omni.kit.livestream.webrtc)

# or self-launch from C#: IsaacSimBridge.LaunchAsync(options) with Gui/MotionBvh/Livestream

# fast inner loop with NO GPU/sim:
C:\isaacsim\python.bat mock/mock_bridge.py    # (or just run the xUnit tests against it)
dotnet test tests/IsaacSimSharp.Tests
```

**Live verification pattern** used throughout (every sensor feature was verified this way):
write a tiny Python ZMQ DEALER client, `new_stage` → add geometry → create sensor → `play` →
`step ~30-60` → `get_sensor_frame`, assert the buffer shape/values, then `shutdown`. Franka asset:
`{GetAssetsRoot}/Isaac/Robots/FrankaRobotics/FrankaPanda/franka.usd` (11 links, 9 DOF).

## 10. Future direction (not built)

The exported USD is the deliberate hand-off artifact for a **native C# digital-twin runtime**:
`ovphysx` (physics; needs new C# P/Invoke over its C API) + `ovrtx` (sensors/render; existing C#
bindings) — no Python, no live Isaac Sim. Keeping `ExportUsdAsync` first-class makes that
transition clean.
